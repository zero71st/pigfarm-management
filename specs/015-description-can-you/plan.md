## Summary
**Primary Requirement**: When a user updates the `PigQty` field on a pig pen, the system must persist the new quantity and automatically recalculate all active (unlocked) `PigPenFormulaAssignment` records by syncing `AssignedBagPerPig` from the source formula's latest `ConsumeRate` and recomputing `AssignedTotalBags` using ceiling rounding.

**Technical Approach**: Extend existing `PigPenService.UpdatePigPenAsync` or `PigPenRepository.UpdateAsync` to detect `PigQty` changes, load related `FeedFormula` entities for each active assignment, update assignment fields atomically, and log the change event for audit.

## Technical Context
**Language/Version**: C# .NET 8  
**Primary Dependencies**: ASP.NET Core 8, Entity Framework Core 8, Npgsql (PostgreSQL provider), SQLite (dev)  
**Storage**: PostgreSQL (production via Railway), SQLite (local dev)  
**Testing**: xUnit, Moq (manual validation per quickstart.md - no automated tests generated by default)  
**Target Platform**: Linux server (Railway), Windows dev  
**Project Type**: Web (Blazor WebAssembly frontend + ASP.NET Core Web API backend)  
**Performance Goals**: <50ms for typical pen update with 5-10 formula assignments  
**Constraints**: Must preserve historical data integrity (locked assignments immutable), atomic updates (all assignments or none), logging for audit trail  
**Scale/Scope**: ~100 pig pens, ~500 formula assignments total, 1-5 concurrent users
